# Obsidian AI Vault — User Guide

## Step-by-Step Setup

### 1. Clone the Repository

```bash
git clone https://github.com/Corin948/obsidian-ai-vault.git
cd obsidian-ai-vault
```

### 2. Open in Obsidian

1. Open Obsidian.
2. Click **"Open folder as vault"**.
3. Select the `obsidian-ai-vault` folder.
4. Install the **Obsidian Git** plugin:
   - Settings → Community Plugins → Browse → Search "Git" → Install "Obsidian Git"
   - Enable the plugin.
5. Configure Obsidian Git:
   - Settings → Obsidian Git → Set backup interval to 10 minutes.
   - Set commit message template to `vault backup: {{date}}`.

### 3. Open in Kimi Code IDE

1. Open Kimi Code (Desktop IDE).
2. File → Open Folder → Select `obsidian-ai-vault`.
3. Kimi Code will automatically read `.kimi/instructions.md` and understand the vault structure.

### 4. Open in Claude Code

1. Open terminal in the vault folder.
2. Run:
```bash
claude
```
3. Claude Code will automatically read `.claude/CLAUDE.md` and know the vault architecture.

## Daily Workflow

### Morning
1. Open Obsidian. Create today's Daily Note (`01-Daily-Notes/YYYY-MM-DD.md`).
2. In Kimi Code, work on any active notes. Use Kimi for writing assistance and inline suggestions.
3. Obsidian Git auto-commits every 10 minutes.

### During Meetings
1. Create a meeting note from template: `Ctrl+N` → Apply `Meeting-Note` template.
2. Paste raw notes or transcript in the "Raw Notes" section.
3. Save. Obsidian Git will push it to GitHub.
4. GitHub Action automatically structures it and extracts action items.

### Evening Review
1. In Claude Code:
```bash
claude "Review today's daily note and all meeting notes. What are my top 3 action items for tomorrow?"
```
2. Claude reads the notes, synthesizes, and gives you priorities.

### Weekly
1. Check `.github/reports/stale_notes.md` (generated by Stale Note Cleaner).
2. Process or delete stale notes.
3. Review the Auto-Linker PR and merge suggested links.
4. In Claude Code:
```bash
claude "Review all notes created this week. Suggest 2 new MOCs if I have emerging themes."
```

## Creating Notes

### From Inbox (Quick Capture)
1. `Ctrl+N` → New note lands in `00-Inbox/` (configured in `.obsidian/app.json`).
2. Write quickly. Don't worry about structure.
3. Within 48 hours: Move to correct folder, add YAML frontmatter, link to MOCs.

### From Template
1. `Ctrl+N` → Apply template from `08-Templates/`.
2. Fill in frontmatter.
3. Write content.
4. Link to relevant MOCs immediately.

### From Claude/Kimi
```bash
claude "Create a permanent note about 'flow state in programming' based on my existing notes about productivity."
```
Claude will:
1. Read relevant MOCs and notes.
2. Create the note in `05-Permanent-Notes/`.
3. Update the relevant MOC.
4. Show you the git diff for review.

## Troubleshooting

### Obsidian Git not committing
- Check that Git is configured with your user name and email.
- Check that the remote URL is correct.
- Manually commit: Command Palette → "Git: Commit all changes".

### GitHub Actions not running
- Check that GitHub Actions are enabled in the repository (Settings → Actions → General).
- Check that the workflow files are in `.github/workflows/`.

### Claude Code not reading vault context
- Ensure you're in the vault root directory when running `claude`.
- Check that `.claude/CLAUDE.md` exists.

### Kimi Code not understanding vault structure
- Check that `.kimi/instructions.md` exists.
- Reload the window: Command Palette → "Developer: Reload Window".

## Customization

### Add a New Project
1. Create folder: `03-Projects/My-New-Project/`
2. Create MOC: `04-MOCs/My-New-Project-MOC.md` (from template)
3. Update `04-MOCs/Master-MOC.md` to link to the new project MOC.

### Add a New Tag
1. Use the tag in a note's YAML frontmatter.
2. Update the relevant MOC to include notes with that tag.
3. Claude Code can audit: `claude "Find all notes tagged 'my-new-tag' and suggest a MOC."`

### Modify Automation
- Edit `.github/scripts/` to change how the Auto-Linker or Meeting Processor works.
- Edit `.github/workflows/` to change triggers or schedules.
